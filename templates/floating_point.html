Números en punto flotante

Demostración de que la computadora no representa exactamente los números reales.

Idea principal:
  • Los números reales se guardan en binario (base 2) con un número finito de bits.
  • Fracciones “bonitas” en base 10, como 0.1 o 0.2, se vuelven infinitas periódicas en base 2.
  • Por eso, 0.1, 0.2 y 0.3 se almacenan como aproximaciones, no como valores exactos.
  • La comparación con == exige igualdad exacta bit a bit, lo cual casi nunca ocurre en estos casos.

Ejemplo obligatorio del profesor (Python puro):

print(0.1 + 0.2 == 0.3)

Forma recomendada en cómputo científico (NumPy):

import numpy as np

a = 0.1
b = 0.2
c = 0.3

print(a + b == c)          # comparación exacta
print(np.isclose(a + b, c))  # comparación con tolerancia
